# PUZZLE TAXONOMY — ARCHIVE OF ECHOES: THE ANUNNAKI LEGACY

> All puzzles are modular, iPhone-touch-native, and reusable across all issues.

---

## Design Principles

1. **Every puzzle must have a narrative reason** — the mechanic IS the meaning
2. **No grind** — one pass solves a puzzle; mastery comes from understanding, not repetition
3. **Lens-variant** — ideally every puzzle has a different flavor per lens
4. **Fail-graceful** — wrong attempts corrupt a panel (increasing difficulty) but never block progress permanently
5. **Touch-native** — optimized for single-thumb use in portrait mode

---

## CATEGORY A — Panel Logic

These puzzles treat panels as physical objects on the page.

---

### A1 | Panel Reorder

**What it is**: A set of panels is presented out of sequence. Player drags them into a correct (or valid) narrative order.

**iPhone gesture**: Long-press to lift a panel, drag to new position, release to place.

**Narrative reason**: History is non-linear. To understand an event, you must sequence causation correctly.

**Variants**:
| Variant | Description |
|---------|-------------|
| Strict order | One correct answer; puzzle validates |
| Flexible order | Multiple valid orders; different orders reveal different captions |
| Circular order | No clear start/end; player must find the "hinge" panel |

**Used in**: Issues 02, 03, 05, 08

**Lens effect**:
- Mythic: panels glow in divine sequence order
- Political: panels show power-transfer logic (who controls whom)
- Symbolic: panels can be ordered by emotional arc, not event order

---

### A2 | Panel Rotation

**What it is**: A panel is rotated to reveal its true orientation. Often solves an alignment or correspondence puzzle.

**iPhone gesture**: Two-finger rotate.

**Narrative reason**: Some truths are only visible from the right perspective.

**Variants**:
| Variant | Description |
|---------|-------------|
| Cardinal align | Rotate until compass markers align (Giza/architecture puzzles) |
| Symbol match | Rotate until a symbol on the panel matches a symbol on an adjacent panel |
| Page-level rotate | Entire page rotated; changes which panels are "above" or "below" others |

**Used in**: Issues 03, 06, 09

---

### A3 | Panel Merge / Split

**What it is**: Two panels are dragged together to "merge" into a single composite insight. Or one panel is split into two — revealing a hidden contradiction.

**iPhone gesture**: Drag two panels toward each other → snap together. Or two-finger spread on a single panel → splits.

**Narrative reason**: Some ideas only make sense together. Others contain hidden contradictions that must be separated.

**Variants**:
| Variant | Description |
|---------|-------------|
| Concept merge | "Anunnaki" + "Atlantis" → "Pre-Diluvian civilization" composite panel |
| Era merge | Two time periods overlap → reveal shared pattern |
| Conflict split | One panel is actually two contradictory readings collapsed together |

**Used in**: Issues 02, 04, 08

---

### A4 | Panel Overlay (Timeline Convergence)

**What it is**: Multiple semi-transparent panels from different eras are stacked. The overlap reveals a pattern hidden in any individual layer.

**iPhone gesture**: Tap panels to add to stack; drag to align overlapping symbols.

**Narrative reason**: The pattern is only visible across eras — no single timeline contains the full picture.

**Primary use**: Issue 08 (Convergence Begins) and Issue 12 (The Last Panel)

**Lens effect**: Each lens changes which elements of each layer are opaque vs transparent.

## CATEGORY B — Corruption & Repair

These puzzles "fix" broken panels to reveal hidden content.

---

### B1 | Stabilize Corruption

**What it is**: A panel is visually glitching (static, distortion, scrambled text). Player holds it steady to stop the chaos, then scrubs to restore it.

**iPhone gesture**:
1. Long-press panel → player "holds reality" (progress bar fills)
2. Slow swipe → scrubs distortion off like cleaning a window
3. Tap damaged sections → "locks" restored lines

**Narrative reason**: Suppressed or contested information distorts. Holding attention restores it.

**Variants**:
| Variant | Description |
|---------|-------------|
| Static removal | Scrub horizontal distortion bands |
| Fragment restore | Tap-to-lock each recovered element in sequence |
| Time decay | Panel corrupts in real-time; player races to stabilize key elements |

**Used in**: All issues (core mechanic; used frequently)

---

### B2 | Ink Restore (Tracing)

**What it is**: A panel has missing linework — sections are blank. Player traces missing lines to complete the image.

**iPhone gesture**: Draw with finger along dotted guide path.

**Narrative reason**: Some records were physically destroyed. Restoration requires patient reconstruction.

**Variants**:
| Variant | Description |
|---------|-------------|
| Guide trace | Dots show the path; player traces it |
| Freehand | No guide; player reconstructs from partial image context |
| Multi-layer | Complete multiple line layers in correct order (each layer changes the meaning) |

**Used in**: Issues 03 (Eve panel), 05, 07

---

### B3 | Defacement Undo (Scribe Interference)

**What it is**: A panel has been censored — key symbols, faces, or words are blacked out. Player uses the Ark Resonance Scan to illuminate and restore them.

**iPhone gesture**: Tap + hold on blacked-out area with Ark equipped → scan reveals underlying content.

**Narrative reason**: Authority erases inconvenient truths. The right tool restores them.

**Used in**: Issues 05, 06, 08 (primarily Scribe interference events)

## CATEGORY C — Geometry & Architecture

These puzzles use spatial logic inspired by architectural and astronomical motifs.

---

### C1 | Cardinal Alignment

**What it is**: A page or panel shows a structure from above. Player rotates the entire page until alignment markers (N/S/E/W indicators) match their correct positions.

**iPhone gesture**: Two-finger rotate the entire page view.

**Narrative reason**: Ancient builders aligned to cardinal points for acoustic/energetic/astronomical reasons (in-game lore). Correct alignment unlocks the structure.

**Variants**:
| Variant | Description |
|---------|-------------|
| True north align | Rotate to match a compass rose marker |
| Star align | Rotate until shaft orientation matches a star map overlay |
| Mirror align | Two structures must reflect each other on aligned axes |

**Used in**: Issues 03, 06, 09

---

### C2 | Shaft Labyrinth

**What it is**: A narrow corridor viewed from a "micro-camera" perspective. Player navigates through tight passages, branch points, and door obstacles to reach a chamber.

**iPhone gesture**: Swipe to advance, tap to interact with obstacles, hold to inspect walls for hidden marks.

**Narrative reason**: The shafts of the Pyramid are not simple channels — they contain sealed doors, encoded marks, and blocked passages.

**Obstacles**:
| Type | Solution |
|------|----------|
| Stone door | Requires specific Ark ability to open |
| Branching path | Direction choice (affects which chamber is revealed) |
| Encoded walls | Read marks → apply in sequence to advance |
| Deadfall (trapped panel) | Player backs out; corruption increases |

**Used in**: Issues 07, 09, 12

---

### C3 | Correspondence Puzzle ("As Above / So Below")

**What it is**: A two-part panel where solving the top half auto-generates the solution to the bottom half (mirroring logic).

**iPhone gesture**: Solve top half normally. Watch bottom half respond. Occasionally must correct bottom half if player's top solution was incomplete.

**Narrative reason**: The Hermetic doctrine of correspondence — every pattern at one scale reflects at another. Used to teach the Lens system.

**Used in**: Issues 04 (Architect arc), 06

---

### C4 | Mirror-City Puzzle

**What it is**: Two city layout panels (representing distant civilizations) must be aligned so shared geometric ratios become visible.

**iPhone gesture**: Drag one layout panel to overlap with another. Slide transparencies until alignment markers light up.

**Narrative reason**: The Architect designed both cities to mirror each other — discovering the correspondence unlocks a Key.

**Used in**: Issue 06 (Mirror Site)

## CATEGORY D — Symbol & Language

These puzzles work with symbol systems, glyphs, and meaning-transfers.

---

### D1 | Glyph Evolution

**What it is**: The same concept is shown expressed in 3 different symbol systems across 3 panels. Player must identify the common meaning and drag it to a "synthesis" slot.

**iPhone gesture**: Examine 3 panels, identify shared concept, drag the matching glyph from each to a central merge point.

**Narrative reason**: Knowledge traveled across cultures by changing its symbol skin while preserving its core meaning.

**Used in**: Issues 04, 07

---

### D2 | Council Decree (Logic Grid)

**What it is**: A council of figures must arrive at a unanimous decree. Player adjusts the arrangement of figures, their stances, and their scroll contents to achieve consensus.

**iPhone gesture**: Drag figures to seats, tap to cycle stances, tap scrolls to change content.

**Narrative reason**: The Council of Fate (Anunnaki archetype) operates by decree — understanding their logic means understanding how power gets encoded into civilization rules.

**Used in**: Issue 02

---

### D3 | Identity Node Puzzle

**What it is**: A single panel shows a figure with two overlapping names. Player adjusts transparency sliders to determine the "dominant" interpretation for their playthrough.

**iPhone gesture**: Dual sliders — left/right names. Adjusting one affects the other. Neither can reach 0 — both "truths" coexist.

**Narrative reason**: Some historical figures exist as narrative composites. The player chooses which reading they carry forward — affecting Resonance Traits and faction relationships.

**Used in**: Issue 05 (Akhenaten / Moses node)

---

### D4 | Semantic Merge / Split

**What it is**: Two word-labels float over a panel. Player either merges them (dragging together) or splits them (spreading apart) depending on what the puzzle calls for.

**iPhone gesture**: Drag two labels toward each other to merge. Two-finger spread to split.

**Narrative reason**: Some ideas are falsely separated. Some are falsely combined. Recognition clarifies meaning.

**Variants**:
| Variant | Description |
|---------|-------------|
| Merge | "Anunnaki" + "Atlantis" → composite understanding |
| Split | "God" → splits into "Ruler" + "Engineer" (lens-dependent) |
| Chain | A cascade merge of 4+ labels into one foundation concept |

**Used in**: Issues 02, 04

## CATEGORY E — Artifact Puzzles (Late Game: Issues 10–12)

---

### E1 | Ark Assembly

**What it is**: Five Ark components are scattered across panels. Player identifies and collects each, then places them in the correct assembly order.

**iPhone gesture**: Tap to collect. In assembly screen: drag component to slot. Snap animation confirms correct placement.

**Components** (fantasy):
1. Acacia chassis (structural frame)
2. Gold overlay (inner) — conductivity
3. Gold overlay (outer) — signal skin
4. Rings and poles (carry constraint)
5. Mercy Seat with two facing figures (interface lid)

**Narrative reason**: The Ark was disassembled to prevent it from being used. Reassembly restores the keystone.

**Constraint mechanic**: Poles must be inserted before mercy seat; overlays must be applied before poles. Wrong order = corruption spike.

**Used in**: Issue 10

---

### E2 | Carry Constraint

**What it is**: Player must navigate the assembled Ark through the Pyramid shaft labyrinth while following the "carry rule" — the Ark cannot be touched directly; only moved via its poles.

**iPhone gesture**: Two-finger tap and drag (simulating two pole-handles). Cannot single-finger drag the Ark body.

**Narrative reason**: Ark lore specifies that direct touch causes catastrophic results (in-game: severe corruption). This becomes a genuine mechanical constraint.

**Used in**: Issues 10, 12

---

### E3 | Circuit Completion

**What it is**: Multiple node points across several panels must be connected by drawing "ink lines" between them. Lines cannot cross. Some nodes require a specific Ark ability to activate first.

**iPhone gesture**: Drag from node to node. Lines drawn. Cannot cross existing lines.

**Narrative reason**: The Pyramid circuit requires all components connected in a specific topology — shafts, rods, Djed, Ark as the final node.

**Node types**:
| Node | Color | Requires |
|------|-------|---------|
| Shaft port | Blue | Panel entry |
| Djed port | Gold | Djed Meter > 50% |
| Ark seat | White | Ark installed |
| Capstone port | Red | Capstone placed |

**Used in**: Issues 11, 12

---

### E4 | Capstone Placement

**What it is**: The final alignment puzzle — the capstone must be oriented precisely and lowered into position. Three simultaneous alignment axes must be satisfied.

**iPhone gesture**:
1. Rotate capstone (two-finger rotate) to match azimuth
2. Tilt phone slightly (gyroscope) OR two-finger slide for elevation alignment
3. Long-press to "lock" when all three axes are satisfied

**Narrative reason**: The capstone is the convergence lock. Incorrect placement leaves the circuit open.

**Used in**: Issue 12 (single use — final major puzzle)

## BONUS: GUTTER PUZZLES (Optional / Secret)

The Gutter holds hidden puzzles that appear only when the player explores the space between panels.

---

### G1 | Gutter Door

**What it is**: A narrow passage in the gutter between two panels. Player slides through it to access a hidden panel or bypass a locked section.

**iPhone gesture**: Swipe along the edge between panels (horizontal or vertical depending on layout).

**Risk**: Frequent Gutter use increases a "Gutter Affinity" trait — affects ending options.

---

### G2 | Gutter Whisper Trace

**What it is**: The Gutter whispers a partial sentence. Player traces the missing words by connecting floating letter fragments.

**Reward**: Reveals a hidden caption that changes interpretation of the current page.
